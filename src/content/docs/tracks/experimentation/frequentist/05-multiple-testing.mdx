Evaluating multiple hypotheses simultaneously inflates the false positive rate beyond the intended significance level. This note derives the mathematical mechanics of this inflation and details the statistical methods used to control it.

## 1. The Multiple Comparisons Problem

When we conduct a single hypothesis test at significance level $\alpha = 0.05$, we accept a 5% probability of a false positive.

With $m$ independent tests, each under their respective null hypotheses:


$$
P(\text{no false positives}) = (1-\alpha)^m
$$




$$
\boxed{P(\text{at least one false positive}) = 1 - (1-\alpha)^m}
$$



This is the **Family-Wise Error Rate (FWER)**. Each additional test consumes part of our overall error budget.

## 2. Practical Scenarios

### 2.1 Multiple Metrics

If a team tests 20 independent metrics at $\alpha = 0.05$:


$$
\text{FWER} = 1 - (0.95)^{20} \approx 0.64
$$



There is a **64% chance** of finding at least one "significant" result even if the treatment does absolutely nothing.

### 2.2 Multiple Variants

Testing $k$ variants against a single control requires $k$ comparisons:

- $k = 5$ button colors: FWER $= 1 - (0.95)^5 \approx 0.23$
- All pairwise comparisons among 5 variants + 1 control: $\binom{6}{2} = 15$ comparisons, FWER $\approx 0.54$

## 3. Structural Mitigations

Before applying statistical corrections, mitigate through experimental design:

- **Define a Primary Metric:** Pre-specify exactly one decision-making metric at $\alpha = 0.05$. Treat all other metrics as guardrails.
- **Avoid Multivariate Testing:** Only pursue it with a pre-specified rationale (e.g., measuring an [Interaction Effect](/gradient-lectures/tracks/experimentation/frequentist/04-interaction-effects/)).

## 4. Statistical Mitigations

### 4.1 Family-Wise Error Rate (FWER) Control

FWER methods maintain $P(\text{at least one false positive}) \leq \alpha$ across all $m$ tests. This is the most stringent form of error control.

#### Bonferroni Correction

To maintain FWER $\leq \alpha$, apply a per-comparison threshold:


$$
\alpha_{\text{pc}} \approx \frac{\alpha}{m}
$$



This follows from the approximation $(1 - \alpha/m)^m \approx e^{-\alpha} \approx 1 - \alpha$ for small $\alpha$.

#### Holm–Bonferroni (Step-Down) Procedure

The Bonferroni correction is inefficient — it applies the same strict threshold to every test regardless of evidence strength. The Holm-Bonferroni method provides the **same FWER guarantee** but is uniformly more powerful.

<Callout type="note" title="Derivation: FWER control under Holm–Bonferroni" collapsible defaultOpen={false}>

Let $m_0$ be the unknown number of true null hypotheses. Suppose $k$ is the index of the *first* true null hypothesis that the procedure rejects. For this to happen:


$$
p_{(k)} \leq \frac{\alpha}{m - k + 1}
$$



Because $k$ is the first true null rejected, all $k-1$ previous rejections were true effects. Therefore $m_0 \leq m - k + 1$, so:


$$
p_{(k)} \leq \frac{\alpha}{m_0}
$$



By Boole's inequality, the probability that *any* of the $m_0$ true null p-values falls below $\alpha/m_0$ is at most $m_0 \times \frac{\alpha}{m_0} = \alpha$. Thus FWER $\leq \alpha$.

</Callout>

**Procedure:**
1. Sort p-values: $p_{(1)} \leq p_{(2)} \leq \cdots \leq p_{(m)}$
2. Reject $H_{(i)}$ if $p_{(i)} \leq \frac{\alpha}{m - i + 1}$
3. Stop at the first failure; accept all remaining nulls

<Callout type="example" title="Holm-Bonferroni worked example" collapsible defaultOpen={false}>

Five tests at $\alpha = 0.05$, sorted p-values $[0.001, 0.008, 0.02, 0.04, 0.08]$:

| Rank | p-value | Threshold $\alpha/(m-i+1)$ | Decision |
|:----:|:-------:|:--------------------------:|:--------:|
| 1 | 0.001 | $0.05/5 = 0.010$ | Reject ✓ |
| 2 | 0.008 | $0.05/4 = 0.0125$ | Reject ✓ |
| 3 | 0.020 | $0.05/3 = 0.0167$ | **Fail — Stop** |
| 4 | 0.040 | — | Accept |
| 5 | 0.080 | — | Accept |

</Callout>

Because Holm-Bonferroni is strictly more powerful, it should always be preferred over plain Bonferroni when FWER control is required.

### 4.2 False Discovery Rate (FDR) Control

FWER control is often too conservative in modern product development. If we test 100 metrics, we may accept 2–3 false positives provided the *proportion* of false positives among our "wins" is low.

FDR controls the expected proportion of false positives among all rejected nulls:


$$
\text{FDR} = \mathbb{E}\!\left[\frac{\text{\# false positives}}{\text{\# total rejections}}\right]
$$



<Callout type="note" title="Derivation: FDR bound" collapsible defaultOpen={false}>

Suppose we have $m$ tests with $m_0$ true nulls. If we reject when $p \leq t$, the expected number of false positives is $m_0 \cdot t$ (since p-values are uniform under the null). Let $R$ be the total rejections. Then:


$$
\text{FDR} \approx \frac{m_0 \cdot t}{R} \leq \frac{m \cdot t}{R}
$$



To guarantee $\text{FDR} \leq \alpha$, we need $t \leq \frac{R}{m}\alpha$. This shows the threshold must scale linearly with the number of discoveries.

</Callout>

#### Benjamini–Hochberg (BH) Procedure

<Callout type="note" title="Derivation: BH step-up procedure" collapsible defaultOpen={false}>

From the FDR bound, we need $t \leq \frac{R}{m}\alpha$. Sort p-values and suppose we reject the first $k$: then $R = k$ and $t = p_{(k)}$. We need:


$$
p_{(k)} \leq \frac{k}{m}\alpha
$$



To maximize power, find the **largest** $k^*$ satisfying this condition and reject all $1, \dots, k^*$.

</Callout>

**Procedure:**
1. Sort p-values: $p_{(1)} \leq \cdots \leq p_{(m)}$
2. Find the largest $k^*$ where $p_{(k^*)} \leq \frac{k^*}{m}\alpha$
3. Reject all nulls for indices $1$ through $k^*$

<Callout type="example" title="BH worked example" collapsible defaultOpen={false}>

Five tests at $\alpha = 0.05$, sorted p-values $[0.008, 0.025, 0.028, 0.06, 0.12]$:

| Rank | p-value | Threshold $k/m \cdot \alpha$ | Pass? |
|:----:|:-------:|:----------------------------:|:-----:|
| 1 | 0.008 | $1/5 \times 0.05 = 0.010$ | ✓ |
| 2 | 0.025 | $2/5 \times 0.05 = 0.020$ | ✗ |
| 3 | 0.028 | $3/5 \times 0.05 = 0.030$ | ✓ |
| 4 | 0.060 | $4/5 \times 0.05 = 0.040$ | ✗ |
| 5 | 0.120 | $5/5 \times 0.05 = 0.050$ | ✗ |

The largest passing rank is $k^* = 3$. Reject null hypotheses 1, 2, and 3 — *including* rank 2 which failed individually. The FDR guarantee is a property of the entire rejection set.

</Callout>

The BH procedure assumes independent or positively correlated tests. It is vastly more powerful than FWER methods for large $m$ — it is the standard choice for screening dozens of secondary metrics.

### 4.3 Syntactic Approaches

There are two mathematically equivalent ways to apply any correction:

1. **Adjust the threshold:** Keep raw p-values, compare against $\alpha/m$ (or BH thresholds).
2. **Adjust the p-value:** Multiply raw p-values by a penalty factor; compare against the standard $\alpha = 0.05$.

Both achieve the same statistical result. Adjusting the p-value is often preferred in tooling because users can continue comparing against the familiar 0.05 threshold.
