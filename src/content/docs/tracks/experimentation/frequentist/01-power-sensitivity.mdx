Proper sizing ensures an experiment runs long enough to reliably detect real business impact, but not so long that it wastes traffic and delays critical product decisions.

## Alpha, Beta, and Power

When comparing the difference in means between a Treatment and Control group, that difference is itself a random variable. We test whether this "delta" is significantly different from zero:

- **Null Hypothesis $H_0$:** The true delta is 0.
- **Alternative Hypothesis $H_A$:** The true delta is non-zero.

In testing these hypotheses, we face two foundational risks:

**Type I Error ($\alpha$):** The probability of rejecting the null when it is actually true.
- *Conceptually:* A **False Positive**. We believe our feature worked, but it actually does nothing.
- Typically bounded at 5% ($\alpha = 0.05$).

**Type II Error ($\beta$):** The probability of failing to reject the null when $H_A$ is true.
- *Conceptually:* A **False Negative**. Our feature was a winner, but our test missed it.
- Typically bounded at 20% ($\beta = 0.20$).

**Statistical Power** ($1 - \beta$) is the probability of correctly detecting a true effect. With $\beta = 0.20$, power is 80%.

## Minimum Detectable Effect (MDE)

The MDE is the smallest true effect size we want to reliably detect given our chosen power and $\alpha$.

- **If set too small:** We require a massive sample size; the test runs for months.
- **If set too large:** The test finishes quickly, but we risk missing smaller improvements that compound into large yearly revenue.

Setting the MDE is a **business decision**, not a statistical one.

## Sample Size Derivation

To understand the constraints, we derive the sample size formula geometrically from the two sampling distributions.

The null distribution $H_0$ is centered at 0. We set a threshold for how extreme our observed delta must be to reject the null — for a two-tailed test, this standardized threshold is $Z_{1-\alpha/2}$.

The alternative distribution $H_A$ is centered at our true effect size. The standardized distance from the $H_A$ center back to the rejection threshold is $Z_{1-\beta}$.

Since both distributions sit on the same axis:

<div class="math-step">

$$
d_{\text{standardized}} = Z_{1-\alpha/2} + Z_{1-\beta}
$$

</div>

De-standardizing by multiplying by the standard error $\text{SE} = \sigma\sqrt{2/n}$ (assuming equal pooled variance and equal group sizes):


$$
d_{\text{real}} = (Z_{1-\alpha/2} + Z_{1-\beta}) \cdot \sigma\sqrt{\frac{2}{n}}
$$



Solving for $n$:

<div class="math-result">

$$
n = \frac{2\sigma^2\left(Z_{1-\alpha/2} + Z_{1-\beta}\right)^2}{d_{\text{real}}^2}
$$

</div>

<Callout type="note" title="Why can we set Z-levels independently?" collapsible defaultOpen={false}>

One might ask: if both distributions sit on the same axis, how can we arbitrarily set $Z_{1-\alpha/2}$ and $Z_{1-\beta}$ independently?

The answer: **the distributions are not static**. In the formula, $n$ is the output. As we change $Z_{1-\alpha/2}$ and $Z_{1-\beta}$, the formula outputs a new $n$. Since $\text{SE} \propto 1/\sqrt{n}$, changing $n$ squeezes or widens both distributions. By computing and using this new $n$, we are effectively shrinking the curves until our hard-coded $Z$ distances perfectly fit within the real-world MDE $d_{\text{real}}$.

</Callout>

## Practical Application

To calculate the required sample size per variant, you need four inputs:

1. **$Z_{1-\alpha/2}$** — e.g., **1.96** for $\alpha = 0.05$
2. **$Z_{1-\beta}$** — e.g., **0.84** for power = 80%
3. **$d_{\text{real}}$** — the target minimum absolute lift (MDE)
4. **$\sigma^2$** — metric variance estimated from historical data (for binary metrics, $\sigma^2 = p(1-p)$)

## Variance Reduction (CUPED)

If traffic is limited and we cannot compromise on MDE, our only mathematical recourse is to reduce the metric variance $\sigma^2$ directly.

**CUPED** (Controlled Experiment Using Pre-Experiment Data) is the industry standard approach.

It leverages the fact that a user's pre-experiment baseline is highly correlated with their in-experiment behavior, and that pre-experiment behavior is completely unaffected by the treatment assignment.

**The adjustment:**


$$
\hat{Y}_i = Y_i - \theta (X_i - \mathbb{E}[X])
$$



where $X_i$ is the user's pre-experiment metric and:


$$
\theta = \frac{\text{Cov}(Y, X)}{\text{Var}(X)}
$$



**The result:** The variance of the adjusted metric is strictly smaller:


$$
\text{Var}(\hat{Y}) = \text{Var}(Y)(1 - \rho^2)
$$



where $\rho$ is the pre/post correlation. If $\rho = 0.8$, variance drops by 64%, allowing significance to be reached in **one-third** of the original sample size.

<Callout type="tip" title="Modern ML Extensions (CUPAC)" collapsible defaultOpen={false}>

While classical CUPED uses a single-variable linear adjustment, modern platforms extend this with machine learning. **CUPAC** (Control Using Predictions As Covariates) trains a gradient-boosted model on hundreds of pre-experiment features (tenure, past clicks, demographics) to generate a highly accurate prediction of $Y$.

The residuals $Y_i - \hat{Y}_{\text{ML}, i}$ are then used as the target metric, driving variance down further than classical OLS approaches. The principle is identical to CUPED — only the covariate is richer.

</Callout>
